1. Правила оформлення програмного коду мовою C
Основні рекомендації з презентації
Читабельність:

Використовуйте осмислені назви змінних і функцій.
Коментуйте складні ділянки коду, пояснюючи "що" і "чому", а не "як".
Іменування:

Для функцій використовуйте стиль CamelCase (наприклад, calculateSum).
Для змінних — snake_case (наприклад, total_sum).
Форматування:

Використовуйте єдиний стиль відступів: 4 пробіли або табуляцію.
Розділяйте логічні блоки порожніми рядками.
Принципи розробки:

Одна функція має виконувати одну задачу.
Функції довжиною понад 30 рядків рекомендується розбивати на менші.
Використання фігурних дужок:

Навіть для однорядкових блоків if використовуйте фігурні дужки.
Структура проєкту:

Назви файлів мають відображати їх призначення (наприклад, main.c, utils.c).
Заголовкові файли (.h) повинні використовуватися для декларації функцій.
Приклади коду
Неправильний код

#include <stdio.h>

void calc() { int a=10, b=20; printf("Sum:%d\n",a+b); }

int main() { if(1) printf("Hello, World!\n"); return 0; }


Правильний код

#include <stdio.h>

// Функція для обчислення суми двох чисел
void calculate_sum(int a, int b) {
    int sum = a + b;
    printf("Sum: %d\n", sum);
}

int main() {
    // Демонстрація використання функції
    calculate_sum(10, 20);

    if (1) {
        printf("Hello, World!\n");
    }

    return 0;
}


Неправильна структура проєкту:
project/
|-- main.c
|-- functions.c

Правильна структура проєкту:
project/
|-- main.c       // Головний файл програми
|-- utils.c      // Реалізація допоміжних функцій
|-- utils.h      // Заголовковий файл для utils.c


Рекомендації:
Коментуйте лише складні ділянки коду.
Дотримуйтесь стандартизованого стилю іменування.
Слідкуйте за структурою коду і файлами в проєкті.
Розробляйте функції, що виконують одну задачу.
